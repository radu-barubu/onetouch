Introduction

This interim release is to provide integrators with an updated version that contains all Nuance SpeechAnywhere Release 1 features. Please be aware that stabilization of these features is ongoing.

Important: This release does not deliver the final Nuance SpeechAnywhere speech recognition quality.

With the release of Beta 1 the alpha version is no longer supported and the server has been switched to the new version.

For more information, browse this SDK documentation or see the end user documentation.

New features since the alpha release

  • Audio codec: Speex wideband has replaced PCM to save bandwidth.
  • The first version of the end user documentation.
  • Capitalization processing has been improved.
  • It is now possible to explicitly exclude user interface controls from being speech-enabled.
  • Additional voice commands:

Correction: scratch that

Navigation: beginning of field, first field, last field, etc.

Advanced field navigation: go to <field name>

  • Improved audio detection.
  • Collection of data to derive key performance indicators (KPIs) and speech recognition analysis
  • Speech-enabled controls are marked visually.
  • Multimodal input (voice, mouse, keyboard) handling has been improved.
  • The feedback bubble has been added.
  • The Recording toolbar has been improved.
  • Mac support has been improved.

Interface changes since the alpha release

  • Topic for VUI controllers (optional): Can be used to associate a topic (area of medicine) per VUI form (speech-enabled UI view).
  • Language for VUI controllers (optional): Can be used to associate a specific language (speech recognition language) per VUI form (speech-enabled UI view).
  • The partner GUID and the license GUID are now be configured via cookie.

Integration changes necessary since the alpha release

  • The password parameter has been removed.
  • Our product name has changed to "Nuance SpeechAnywhere". This has resulted in a variety of API changes.

Known issues in Nuance SpeechAnywhere Beta 1

  • In Internet Explorer the Nuance logo indicating that fields are speech-enabled does not display.
  • The Nuance logo in speech-enabled fields is covered when the field has a scroll bar.
  • Citrix is not yet fully supported.
  • In Microsoft Internet Explorer if the user pauses to wait for the recognized text to be displayed and then says new paragraph, the text continues on the next line and no blank line is left.
  • It is not possible to start a speech recognition session for a user name that contains special characters.
  • There are problems with the recognition of voice commands.
  • There are problems with the recognition of advanced field navigation voice commands.
  • The feedback bubble might not display correctly in some situations. The feedback bubble volume meter sensitivity is not yet final.
  • In Microsoft Internet Explorer recording cannot be stopped from the feedback bubble.
