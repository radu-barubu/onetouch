Introduction

With the release of Nuance SpeechAnywhere 1.0 SR 1 the previous version is no longer supported and the server has been switched to the new version.

For more information on all the features, browse this SDK documentation or see the end user documentation.

New features

Dynamically modified controls: When the GUI view is changed after the VUI form has been created by Nuance SpeechAnywhere, call the NUSA_reinitializeVuiForm() method. The VUI form available to the user is adjusted the next time the user clicks the Record button.

Back to Top

Interface changes

Any microphone which can create 16-bit and at least 16 kHz audio data is supported. If your users are using microphones that do not create audio data of good enough quality for speech recognition a message box is displayed; they can use your application but not with speech recognition.

Back to Top

Known issues

  • Nuance SpeechAnywhere does not work with the MacBook Pro internal microphone.
  • The Nuance flame in speech-enabled controls is covered when the control has a scroll bar in the following browsers: Microsoft Internet Explorer 8, Google Chrome 9, Mozilla Firefox 3, Apple Safari 5.
  • If you are using Firefox make sure to click Yes in the Microsoft Silverlight Configuration dialog box; if you click No Firefox might stop responding.
  • If recording does not work an error message about changing the default device is displayed. However, changing the default device might not solve the problem.
