Release Notes for Nuance SpeechAnywhere 1.1

With the release of Nuance SpeechAnywhere 1.1 the server has been switched to the new version.

This release includes several fixes. No integration changes are required on your part to access the improvements.

For more information on all the features, browse this SDK documentation or see the end user documentation.

Enhancements

  • We now support the HTML5 contenteditable attribute for Microsoft Internet Explorer and Mozilla Firefox browsers to create plain text content.
  • It is now possible to dynamically change the set of speech-enabled controls (for more information, see: NUSA_reinitializeVuiForm).
  • It is now possible to configure whether Nuance SpeechAnywhere actively speech-enables all controls or you explicitly tag the controls you want speech-enabled (for more information, see: NUSA_enableAll).

Interface changes 

Any microphone which can create 16-bit and at least 16 kHz audio data is supported. If your users are using microphones that are not able to create audio data of at least this quality a message box is displayed; they can use your application but not with speech recognition.

Known issues

  • Microsoft Silverlight known issue: Nuance SpeechAnywhere does not work reliably on Mac OS. Microsoft are considering fixing the problem in an upcoming version of Silverlight.
  • Mozilla Firefox known issue: When the web page is re-loaded in Firefox, recorded text is moved from the field it was originally dictated into to the next field on the page. Mozilla are aware of the problem
  • The Nuance flame in speech-enabled controls is covered when the control has a scroll bar in the following browsers: Microsoft Internet Explorer 8, Google Chrome 9, Mozilla Firefox 3, Apple Safari 5.
  • Sometimes after adding a new text control during recording (via NUSA_reinitializeVuiForm), recognition does not work properly.
