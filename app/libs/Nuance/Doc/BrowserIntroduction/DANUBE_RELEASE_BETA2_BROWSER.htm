Introduction

This interim release is to provide integrators with an updated version that contains most of the Nuance SpeechAnywhere Release 1 features. Please be aware that stabilization of these features is ongoing.

Important: This release does not deliver the final Nuance SpeechAnywhere speech recognition quality.

With the release of Nuance SpeechAnywhere Beta 2 the beta 1 version is no longer supported and the server has been switched to the new version.

For more information, browse this SDK documentation or see the end user documentation.

New features since the beta 1 release

  • You can customize the appearance of the button the user clicks to start and stop recording; for more information, see: Customize the start/stop recording icon.
  • You can customize the background color of the speech-enabled control that has the speech focus; for more information: see: Customize the speech focus style.

Interface improvements since the beta 1 release

  • Formatting now differentiates between a new line and a new paragraph. At a new line the text starts at the beginning of the next line and at a new paragraph the next line remains empty and the text starts at the beginning of the following line.
  • Nuance SpeechAnywhere now supports PowerMic on MacOS.

Known issues in Nuance SpeechAnywhere Beta 2

  • Recording is not always picked up properly; the feedback bubble volume meter sensitivity is not yet final. If the microphone recording volume is too high the user might experience decreased recognition performance. We recommend lowering the volume setting.
  • In Microsoft Internet Explorer if your web page is https a warning is displayed about secure and non-secure items. This is because some of the Nuance SpeechAnywhere graphics are hosted via http.
  • On web pages based on Microsoft ASP.NET technology the Nuance SpeechAnywhere feedback bubble and speech focus style for speech-enabled controls do not display. The start/stop recording icon is displayed and speech recognition works properly.
  • The Nuance logo in speech-enabled controls is partly covered when the control has a scroll bar.
  • Silverlight might crash if the web page is continuously open for some time.
