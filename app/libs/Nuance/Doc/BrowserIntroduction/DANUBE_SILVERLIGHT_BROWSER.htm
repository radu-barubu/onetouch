Microsoft Silverlight

Nuance SpeechAnywhere (Browser edition) uses a Silverlight-based component for audio recording. It will be compatible with the latest released major version of Silverlight (from version 4 on). For more information, see: www.microsoft.com/silverlight.

Deployment personnel must be made aware of the requirements and the steps necessary for each server and user.

Installing Silverlight

Silverlight must be installed on every client, for every supported browser and operating system, that is speech-enabled. If it is not, your web page will be usable but speech recognition functionality will not be available. If necessary, the user can install Silverlight directly from their browser (if they have sufficient rights to install software).

For more information, see: www.microsoft.com/getsilverlight/get-started/install.

Updating Silverlight

  • Microsoft Windows and MacOS: To configure update settings via the browser, right-click and select Silverlight. In the Microsoft Silverlight Configuration dialog box open the Updates tab.
  • Microsoft Windows: To update automatically, use Windows Automatic Updates in the Control Panel.

Enterprise deployment/update

The Silverlight installers for Microsoft Windows and MacOS can be deployed via appropriate deployment mechanisms available for each platform. For more information, see: 'Silverlight Enterprise Deployment Guide' (Silverlight_Deployment_Guide.docx available from www.microsoft.com/silverlight/resources).

Compatibility

Silverlight versions earlier than version 4 do not support audio recording. If an older, unsupported version of Silverlight is installed on a client, the user will be asked to update to a newer version.

When Microsoft releases a new version of Silverlight, Nuance will test with this version and update SpeechAnywhere as necessary to retain compatibility.

Configuring Silverlight

Each user must configure microphone access the first time a speech-enabled page from a specific server is loaded; this setting is applied to all pages from this server.

If access is not allowed the user is not asked again; the web page is usable but speech recognition functionality will not be available.

To change this setting from the browser, right-click and select Silverlight. In the Microsoft Silverlight Configuration dialog box open the Permissions tab. .

Note: Other settings (for example, selecting a preferred audio device) can also be made in this dialog box.

Troubleshooting

  • The Microsoft Silverlight Configuration dialog box can also be started as a separate application on both Windows and MacOS (go to the Start menu, All Programs, Microsoft Silverlight).

  • Audio recording can be enabled manually (if the standard dialog box does not appear for a web page):

Windows operating systems: Via the Registry Editor (HKEY_CURRENT_USER\Software\AppDataLow\Software\Microsoft\Silverlight\Permissions\...).

MacOS: Edit the Settings -> Permissions keys in the Silverlight .plist file: LibraryPreferences com.microsoft.silverlight.plist:

  • When an end user opens your application and Silverlight is not installed the recording button is replaced with an Install Silverlight button. Be aware that if you are using the standard Nuance recording icon it is smaller than the Install Silverlight button. Make sure to allow enough space for the Install Silverlight button to be properly displayed so that your end users can see the button label properly.
