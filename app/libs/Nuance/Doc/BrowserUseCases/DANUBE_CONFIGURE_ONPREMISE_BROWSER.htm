Configuring Nuance SpeechAnywhere for on-premise deployment

To configure Nuance SpeechAnywhere clients to connect to the Nuance SpeechAnywhere Server set up within your organization’s network, you must pass the server URL and resource URL to the client before opening a client session. You can use the NUSA_ServerURL cookie and NUSA_ResourceURL cookie to do this. Make sure that you know the URL of the Nuance SpeechAnywhere Server. For more information about configuring a Nuance SpeechAnywhere Server, see the Nuance SpeechAnywhere Administration Guide.

The URL you pass to the Nuance SpeechAnywhere client consists of the server URL and the additional query string mode=1. If you do not pass the query string, the server will reject the connection attempt.

For example:

Nuance SpeechAnywhere Server URL: https://your.server.com:1234

URL passed to the client: https://your.server.com:1234/?mode=1

The resource URL should point to the server and location where the Nuance SpeechAnywhere browser files are hosted (css, script, etc.).

Prerequisites

The Nuance SpeechAnywhere Server is installed and configured.

Procedure

To connect your clients to the Nuance SpeechAnywhere Server, set the NUSA_ServerURL cookie according to the Nuance SpeechAnywhere Server URL and add the query string mode=1.

For example: NUSA_ServerURL=https://<your.server.com>:1234/?mode=1

To reference the Nuance SpeechAnywhere resources, set the NUSA_ResourceURL cookie according to the web server hosting Nuance SpeechAnywhere.

For example: NUSA_ResourceURL=https://<your.server.com>

Then continue speech-enabling your web pages, see. Speech-enabling your web browser application.
