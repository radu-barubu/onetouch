Speech-enabling your web browser application

By default all <input> and <text area> controls on your web page are speech-enabled.

Nuance SpeechAnywhere automatically provides a button to start recording on web pages that are initialized for speech recognition:

To change the position or appearance of the record button on your web page, see: Record button.

Note: If your application has sections that you do not want to be speech-enabled you can integrate Nuance SpeechAnywhere by only speech-enabling specific controls; see below.

Procedure

To initialize speech recognition for a web page, do the following:

  1. Set the NUSA_Guids cookie according to the partner GUID and license GUID you receive with the license information:

    2. NUSA_Guids=<license GUID>/<partner GUID>

  2. Copy the following into your web page and set the parameters:

    3. <link rel="stylesheet" type="text/css" href="https://speechanywhere.nuancehdp.com/1.1/css/Nuance.SpeechAnywhere.css" />

    <script type="text/javascript" src="https://speechanywhere.nuancehdp.com/1.1/scripts/Nuance.SpeechAnywhere.js"></script>

    <script language="javascript" type="text/javascript">

    function NUSA_configure() {

NUSA_enableAll = "true";

    NUSA_userId = "<user name>";

    NUSA_applicationName = "<your application name>";

    }

    </script>

When your web page is loaded it is now capable of speech recognition.

Note: Make sure to do this for every web page in your application that you want to be speech-enabled.

Remarks

User names and application names can have a maximum of 75 characters.

Optional parameters

  • You can set specific controls not to be speech-enabled; add the attribute data-nusa-enabled to each of these controls and set its value to "false".
  • You can enable specific controls for named field navigation (using a voice command to move directly to a specific field; for example, medical history); add the attribute data-nusa-concept-name to each of these controls and set its value to the label of the field. This should be a label similar to what the user says when navigating to that field, do not use abbreviations or book title capitalization; for example, data-nusa-concept-name="medical history".
  • By default the speech-enabled control with the speech focus has a light green background. The background remains green while speech recognition processing is ongoing in that control, even if the speech focus is moved to another speech-enabled control.

You can customize this style in your stylesheet. Define your own style for the NUSA_focusedElement class. For example:

.NUSA_focusedElement

{

background: #CCFFCC;

color: #363636;

}

  • In order to ensure that the feedback bubble is properly displayed, the z-index of the NUSAI_Bubble_Container is set to 9000. You can override this setting in your stylesheet. Set the value as follows:

#NUSAI_Bubble_Container{

z-index: <your value>;

}

Speech-enabling only specific controls

To have sections of your application that are not speech-enabled, do the following:

  1. In the NUSA_configure function change the NUSA_enableAll property to "false". None of the controls on your web page are speech-enabled.
  2. Add the attribute data-nusa-enabled to the specific controls that you want to be speech-enabled and set its value to "true".
